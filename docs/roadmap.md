# 项目路线图

本文档记录 `chart-agent` 从当前可展示原型走向更完整产品形态的阶段规划。路线图不等同于承诺排期，用于帮助协作者理解优先级。

## 当前阶段

当前项目处于“可展示原型”阶段，已经完成：

- CopilotKit 原生聊天入口。
- Node Runtime + FastAPI + React 三服务链路。
- 后端 Agent 主导的图表生成、编辑和当前图表问答。
- 受控 `ChartAgentAction` 协议。
- 受控生成式 UI `uiBlocks`。
- mock 指标服务和基础测试体系。
- 中文工程规范、公开 README 和展示截图。

## 短期目标

优先保证公开项目可读、可跑、可解释：

1. 完成公开前检查，确认 README、截图、文档索引和默认启动链路准确。
2. 稳定 E2E 测试，降低 CopilotKit 请求等待带来的偶发时序抖动。
3. 继续拆分过大的后端聚合文件，并细化前端 UI Blocks 组件边界，降低后续维护成本。
4. 补充真实数据源接入设计，不直接把数据库逻辑写进 Agent。
5. 明确许可证策略。

## 中期目标

围绕真实业务数据和更复杂图表能力扩展：

1. 接入真实指标目录或语义指标服务。
2. 支持数据源适配器，例如 REST API、数据库只读查询服务或 BI 平台。
3. 增加多指标、多维度和更复杂筛选表达。
4. 增强当前图表问答，支持排序、占比、环比、Top N 等分析问题。
5. 增加图表持久化和分享能力。
6. 增加更丰富的受控 UI Blocks，例如异常提示、对比卡、过滤器摘要、操作确认卡。

## 长期目标

面向产品化和生产部署：

1. 支持多图 dashboard。
2. 支持用户登录、租户隔离、权限和审计。
3. 支持生产级部署、日志、监控、限流和错误追踪。
4. 引入 CI/CD，在项目足够稳定后自动跑单测、构建和 E2E。
5. 逐步引入后端节点级流式状态，而不是只由 Runtime 基于最终结果生成步骤快照。
6. 建立稳定的插件式数据源和图表能力扩展机制。

## 暂不计划

以下方向暂不作为近期重点：

- 让 LLM 直接生成 React 代码。
- 让 LLM 直接生成任意 ECharts option。
- 让 Agent 直接拼接 SQL 查询真实数据库。
- 在前端注册业务型 `useFrontendTool` 执行图表创建和修改。
- 未经过协议校验就应用模型输出。
